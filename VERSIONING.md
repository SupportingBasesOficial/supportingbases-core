SupportingBases Core - Versioning Policy

This project follows Semantic Versioning 2.0.0.

MAJOR: incompatible API changes, invariant violations, or formula changes that break consumers.
MINOR: additive features and outputs that do not break existing contracts.
PATCH: bug fixes and non-behavioral changes.

SBInput and SBOutput are frozen under v1.0.0. Any change to these contracts requires a MAJOR version bump.
